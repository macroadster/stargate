Agents System

Built-in Go agents for autonomous steganographic analysis and task execution.

System Status

● Active Agents engine running

Poll interval: 60s | Executor: placeholder (stub)

Components

Orchestrator Go

Central coordinator that manages the agent lifecycle. Watches for pending tasks, dispatches work to the worker pool, and collects results.

  • File: orchestrator.go
  • Watcher: Polls for new tasks every STARGATE_AGENT_POLL_INTERVAL
  • Worker: Executes tasks using the configured Executor interface

Worker Go

Runs tasks submitted by the orchestrator. Supports pluggable executors — the default stub produces placeholder reports.

  • File: worker.go
  • Executor: Implement agents.Executor to connect real LLM-driven work
  • Results: Written to UPLOADS_DIR/results/<hash>/

Watcher Go

Monitors the task queue and triggers the orchestrator when new work is available.

  • File: watcher.go
  • Config: Enable via STARGATE_AGENT_WATCHER_ENABLED=true

State & Config Go

Manages agent configuration and runtime state persistence.

  • Files: state.go, config.go, types.go
  • Executor: executor.go — default PlaceholderExecutor
